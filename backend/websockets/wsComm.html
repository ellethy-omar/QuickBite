WebSocket API Documentation

Endpoints

  • /driver: Driver → location updates, order accept
  • /user: User → chat & order notifications
  • /admin: Admin → all real‑time broadcasts
  • /restaurant: Restaurant → new orders, accept/deny

Connecting

Open a socket to:

ws://<host>:<port>/<endpoint>?token=<JWT>

Driver > Server

{
  "type": "locationUpdate",
  "data": { "orderId": "abc123", "latitude": 30.04, "longitude": 31.23, "userId": "user789" }
}
{
  "type": "acceptOrder",
  "data": { "orderId": "abc123", "userId": "user789" }
}

Server > User

{
  "type": "locationUpdate",
  "data": { "orderId": "abc123", "latitude": 30.04, "longitude": 31.23 }
}
{
  "type": "orderAccepted",
  "data": { "orderId": "abc123", "driverId": "driver456", "eta": 10 }
}

User > Server

{
  "type": "chat",
  "data": { "toId": "driver456", "toRole": "driver", "message": "Hi, where are you?" }
}

Admin

Server > Admin

{
  "type": "locationUpdate",
  "data": { "orderId": "abc123", "latitude": 30.04, "longitude": 31.23 }
}

Admins receive every driver’s locationUpdate broadcast.

Restaurant

Server > Restaurant

{
  "type": "newOrder",
  "data": {
    "orderId": "abc123",
    "items": [ /* … */ ],
    "userId": "user789"
  }
}

Restaurant > Server

{
  "type": "respondOrder",
  "data": { "orderId": "abc123", "accept": true, "userId": "user789" }
}

If accept is false, the restaurant has denied (e.g. too many open orders).

Notes

  • All messages are JSON.
  • Use ping/pong to detect disconnects.
  • Switch to wss:// in production.
